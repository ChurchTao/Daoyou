# 炼体系统重设计方案

## 1. 文档状态

- 状态：方案稿
- 日期：2026-06-17
- 适用范围：炼体轨道、丹药推进、角色状态、battle-v5 派生属性、突破承压、副本连续作战、前端展示
- 目标：将当前“炼体等价于永久属性 +1”的低感知系统，升级为独立于境界基础属性的横向成长系统
- 非目标：本方案不重做完整修为境界系统、不替换 battle-v5、不重做全部炼丹系统、不把炼体设计成独立职业系统

## 2. 当前问题判断

当前炼体实现存在三个根本问题。

第一，炼体奖励直接写入 `Cultivator.attributes`。早期 `TrackConfigRegistry` 中，五条炼体轨道升级后分别给 `vitality / spirit / wisdom / speed / willpower +1`，阈值为 `100 * (level + 1)`。这让炼体和境界突破成长竞争同一组基础属性。当前实现已改为新五轨派生效果，炼体五轨升级阈值为 `100 + 70 * level`。

第二，收益数值没有独立乘区。境界阶段属性上限随大境界快速增长，`+1` 基础属性在中后期会迅速变得不可感知。更严重的是，突破成长本身也会增加基础属性，玩家很难区分“这是突破成长”还是“之前炼体积累的长期收益”。

第三，轨道命名太像五维属性的影子，缺少修仙小说中的身体结构感。玩家看见“炼体·体魄、炼体·灵力、炼体·悟性”时，很难理解自己到底在淬炼什么，也难形成“体修体系”的幻想。

结论：炼体不应该继续作为“永久属性丹的包装”。它应该成为境界之外的肉身横向成长层，负责防御、承压、恢复、爆发、持久、抗夺舍和渡劫稳定性。

## 3. 设计原则

### 3.1 与境界基础属性分层

境界突破只改角色的基础成长层。炼体只改肉身修正层。

目标公式：

```text
最终战斗面板 =
  境界基础属性
  + 功法 / 法宝 / 神通修正
  + 炼体修正
  + 当前状态修正
```

炼体收益不再直接写入 `Cultivator.attributes`。炼体等级应存入 `condition.tracks.bodyCultivation` 或等价结构，由 display/combat adapter 在运行时转成 battle-v5 modifier。

### 3.2 小级有感，大级质变

炼体每一级必须有可见收益，但不能只靠线性堆数值。推荐结构：

- 每级：稳定百分比或小额派生加成
- 每 5 级：节点被动
- 每个肉身大阶：机制质变

### 3.3 轨道有分工，不做同质化五属性

新五轨：

- 皮肤
- 筋骨
- 脏腑
- 气血
- 元神

这五轨不是旧五维属性的别名，而是五个身体系统。每轨都要回答三个问题：

- 小说理论上修什么
- 战斗中解决什么问题
- 游戏系统中接哪些模块

### 3.4 有代价，有风险，有材料闭环

炼体不是安全吃丹堆数值。成熟体修体系应有代价：

- 猛烈淬炼会带来丹毒、暗伤、脏腑负荷、气血亏空
- 破阶失败会留下状态或进度损耗
- 高阶炼体需要材料、丹药、副本试炼和境界条件

但首版落地应避免过重惩罚，先让正反馈成立，再逐步接入风险。

## 4. 新五轨设计

### 4.1 皮肤：皮膜筋膜，外护层

#### 小说理论

皮肤不是单纯表皮，而是皮膜、皮下组织和筋膜构成的外护层。它通过药浴、雷击、风刃切割、寒煞侵蚀等方式反复淬炼，使皮膜致密、筋膜韧化。

大成者皮膜如龙鳞，寻常刀剑难伤，且能隔绝毒气、瘴气和魔气入侵。若外护层不固，后续筋骨和脏腑再强，也可能被透劲、暗毒或侵蚀效果直接攻破。

#### 游戏定位

皮肤轨道负责“第一层防线”。

核心关键词：

- 物理防御
- 固定减伤
- 侵蚀抗性
- 中毒 / 瘴气 / 魔气抵御
- 减少战斗后轻伤概率

#### 推荐收益

每级收益：

- 受到的直接伤害降低 `0.6%`
- 中毒、腐蚀、瘴气类持久状态命中率降低 `0.8%`

每 5 级节点：

- 进入战斗时获得一层“外膜护体”：吸收 `2% 最大气血` 的伤害
- 若本场战斗受到的单次伤害低于最大气血 `3%`，有概率完全抵消该次伤害

大阶质变：

- 铜皮：固定伤害减免
- 铁膜：中毒和腐蚀持续回合减少
- 龙鳞：战斗开始获得可刷新的护盾
- 无漏皮膜：部分环境侵蚀不再直接生成持久状态

#### 系统接入

- battle-v5：减伤、护盾、抗异常命中
- condition：毒 / 瘴 / 魔气类长期状态抗性
- dungeon：环境侵蚀事件的减免
- alchemy：药浴、外敷类炼体丹主要推进此轨

### 4.2 筋骨：筋骨骨髓，架构层

#### 小说理论

筋骨是肉身的承重结构，包含硬骨、筋腱、关节和骨髓。硬骨决定力量杠杆，骨髓决定造血与恢复潜能。

修炼方式包括吸纳玄铁精、妖兽骨髓、地磁重压、重力挤压，使骨质玉化、骨髓如汞。强者举鼎拔山只是表象，更关键的是骨髓造血速度极快，伤口愈合、断骨续接、甚至断肢重生都来自此层。

#### 游戏定位

筋骨轨道负责“承重、力量和恢复底盘”。

核心关键词：

- 防御上限
- 暴击伤害减免
- 受伤后恢复
- 重伤降级
- 肉身力量

#### 推荐收益

每级收益：

- 最大气血 `+0.8%`
- 受到暴击伤害降低 `0.8%`
- 自然气血恢复效率 `+0.6%`

每 5 级节点：

- 战斗结束后，若当前气血低于 `50%`，额外恢复 `3% 最大气血`
- 战斗后伤势判定有概率降一级，例如 `major_wound -> minor_wound`

大阶质变：

- 铁骨：暴击伤害减免
- 玉骨：重伤恢复时间缩短
- 汞髓：战斗结束自动恢复一段气血
- 不灭骨：濒死后保留 1 点气血并进入冷却

#### 系统接入

- battle-v5：最大气血、暴击减伤、濒死保护
- condition：自然恢复、伤势降级
- breakthrough：突破失败时伤势减轻
- dungeon：连续战斗续航

### 4.3 脏腑：五脏六腑，动力层

#### 小说理论

脏腑是炼体中风险最高、收益最大的动力层。外在筋骨再强，如果五脏不能承载高压运转，体修爆发时会先把自己震碎。

修炼方式包括采天地五行之气滋养心、肝、脾、肺、肾，或以内家真火煅烧。脏腑强大者能短时间超负荷运转，燃烧气血提升战力而不崩碎。五气朝元也是口吐剑气、肺腑雷鸣、音波功等肉身神通的前提。

#### 游戏定位

脏腑轨道负责“爆发和神通承载”。

核心关键词：

- 短时爆发
- 技能 / 神通承载
- 超负荷代价降低
- 法力 / 气血转换效率
- 五行抗性

#### 推荐收益

每级收益：

- 造成伤害 `+0.5%`
- 技能消耗造成的自身资源压力降低 `0.5%`
- 五行环境 / 五行异常抗性 `+0.5%`

每 5 级节点：

- 每场战斗首次释放高消耗技能后，返还部分法力或气血
- 低气血时获得短暂伤害增幅，但有冷却

大阶质变：

- 五脏初凝：技能消耗降低
- 五气归元：战斗中可触发一次爆发
- 雷音肺腑：解锁音波 / 震慑类肉身神通前置
- 真火炼脏：燃血爆发的负面代价降低

#### 系统接入

- battle-v5：伤害增幅、消耗返还、低血爆发
- creation-v2：部分肉身神通或脏腑系 affix 需要此轨等级
- condition：超负荷后的短期状态
- alchemy：五行药性丹主要推进此轨

### 4.4 气血：气血精元，能量层

#### 小说理论

气血是无形之体，是炼体区别于凡人武夫的标志。它不是普通血液，而是肉身活动的高密度燃料。

修炼方式包括吞噬天材地宝、炼化妖兽精血，将能量压缩于周身穴窍。气血旺盛者寿命远超同阶，体型可随意收缩变化。气血枯竭则会瞬间衰老，根基崩塌。

#### 游戏定位

气血轨道负责“战斗持久力和肉身资源池”。

核心关键词：

- 最大气血
- 战斗续航
- 寿元
- 气血燃烧
- 连续作战

#### 推荐收益

每级收益：

- 最大气血 `+1.2%`
- 战斗后气血保留 / 恢复效率 `+0.8%`
- 丹毒对自然恢复的压制降低 `0.3%`

每 5 级节点：

- 副本连续战斗后额外恢复气血
- 气血低于阈值时，下一次恢复效果提高

大阶质变：

- 血如铅汞：最大气血显著提高
- 气血狼烟：低血时攻击和恢复并行
- 穴窍藏精：战斗开始获得气血储备
- 法天象地：高阶体型变化 / 爆发形态前置

#### 系统接入

- battle-v5：最大气血、战后恢复、低血恢复放大
- condition：当前 HP、自然恢复、丹毒恢复惩罚
- lifespan：少量寿元或寿元损耗抗性
- dungeon：连续作战耐久

### 4.5 元神：元神识念，统御层

#### 小说理论

高阶体修必须炼神，否则只是空有蛮力的躯壳。元神轨道不是法修专属，而是让神魂与肉身每一寸细胞融合，达到身神合一。

元神圆满者无需肉眼视物，周身百丈内纤毫毕现；能抵抗夺舍、魅惑、心魔。渡劫时，元神负责驾驭肉身硬扛天雷，避免肉身和神魂产生排异反应。

#### 游戏定位

元神轨道负责“控制、豁免和高阶承载”。

核心关键词：

- 控制抗性
- 心魔抵抗
- 夺舍 / 魅惑 / 幻术豁免
- 渡劫稳定性
- 高阶肉身神通控制力

#### 推荐收益

每级收益：

- 控制抗性 `+0.8%`
- 暴击抗性或神魂类伤害减免 `+0.5%`
- 突破失败产生心魔的概率降低 `0.4%`

每 5 级节点：

- 每场战斗首次受到控制时，缩短控制持续时间
- 突破失败时，降低 `inner_demon` 或精神类持久状态生成概率

大阶质变：

- 身神相照：控制持续时间降低
- 神识化触：命中 / 识破类加成
- 魂肉合一：夺舍、魅惑、心魔抗性提高
- 驭劫真神：渡劫或大境界突破失败惩罚降低

#### 系统接入

- battle-v5：控制抗性、控制持续时间、暴击抗性
- breakthrough：心魔、清心、护脉等状态交互
- dungeon：幻境、夺舍、神魂侵蚀事件抗性
- creation-v2：高阶肉身神通前置条件

## 5. 肉身大阶设计

五轨提供横向成长，但玩家还需要一个总览性目标。建议增加“肉身境界”，由五轨总等级和关键轨道最低等级共同决定。

### 5.1 阶位建议

| 肉身阶位 | 定位 | 推荐解锁阶段 |
|---|---|---|
| 凡躯 | 初始肉身 | 炼气 |
| 铜皮 | 外护初成 | 炼气 / 筑基 |
| 铁骨 | 承重成型 | 筑基 |
| 玉髓 | 恢复和承压质变 | 金丹 |
| 金身 | 体修体系成型 | 元婴 |
| 法身 | 肉身承载神通 | 化神 / 炼虚 |
| 道体 | 渡劫级肉身 | 合体以上 |

### 5.2 晋阶规则

不建议只看总等级，否则玩家可以单堆一条轨道。推荐规则：

```text
可晋升下一肉身阶位 =
  五轨总等级达到要求
  AND 至少三条核心轨道达到最低等级
  AND 当前大境界满足上限
  AND 完成一次肉身破限操作
```

例如：

- 铜皮：任意三轨 Lv.3，总等级 12
- 铁骨：筋骨 Lv.8，皮肤 Lv.6，总等级 30
- 玉髓：筋骨 Lv.12，气血 Lv.10，总等级 55
- 金身：五轨最低 Lv.10，总等级 90
- 法身：脏腑 Lv.18，元神 Lv.18，总等级 140
- 道体：五轨最低 Lv.25，总等级 220

### 5.3 晋阶收益

肉身阶位不只提供数值，应提供系统权限：

- 铜皮：开启环境侵蚀减免
- 铁骨：开启战后伤势降级
- 玉髓：开启濒死保护
- 金身：开启燃血爆发
- 法身：开启肉身神通承载
- 道体：开启渡劫承压加成

## 6. 数值模型

### 6.1 为什么不用固定属性

固定属性在当前游戏中有三个缺点：

- 容易被境界成长稀释
- 容易被阶段 cap 吸收
- 玩家看不到来源

因此炼体主收益应使用派生 modifier，优先选择：

- 百分比最大气血
- 伤害减免
- 暴击伤害减免
- 控制抗性
- 自然恢复倍率
- 战后恢复
- 状态生成概率降低

这些收益既有感，又不直接污染基础属性。

### 6.2 推荐首版数值

首版数值应保守，但必须可感知。建议每条轨道 1 级的收益大约等于“一个很小的装备副词条”，5 级节点约等于“半个普通被动”。

| 轨道 | 每级收益 | 5 级节点 |
|---|---|---|
| 皮肤 | 直接伤害减免 +0.6%，侵蚀抗性 +0.8% | 入场护盾 2% 最大气血 |
| 筋骨 | 最大气血 +0.8%，暴击伤害减免 +0.8% | 战后低血额外恢复 |
| 脏腑 | 造成伤害 +0.5%，技能资源压力 -0.5% | 高消耗技能返还资源 |
| 气血 | 最大气血 +1.2%，战后恢复 +0.8% | 副本连续战斗恢复 |
| 元神 | 控制抗性 +0.8%，心魔概率 -0.4% | 首次控制持续时间降低 |

### 6.3 上限控制

为了防止炼体压过境界系统，收益需要按肉身阶位设置软上限。

示例：

| 肉身阶位 | 单轨等级软上限 | 总收益上限倾向 |
|---|---:|---|
| 凡躯 | 5 | 教学期 |
| 铜皮 | 10 | 轻度可感知 |
| 铁骨 | 15 | 同境界优势 |
| 玉髓 | 22 | 可支撑体修打法 |
| 金身 | 30 | 明显横向构筑 |
| 法身 | 40 | 神通承载 |
| 道体 | 55 | 渡劫承压 |

软上限不是禁止继续推进，而是超过当前肉身阶位后，进度收益降低，直到完成破限。

## 7. 丹药与材料设计

### 7.1 丹药分类

炼体丹应分为三类。

#### 温养类

低丹毒、稳定推进，适合日常。

例：

- 润膜散：皮肤经验 +40，丹毒 +3
- 壮骨丸：筋骨经验 +40，丹毒 +4
- 养脏丹：脏腑经验 +35，丹毒 +5

#### 猛炼类

高推进、高风险，适合短期冲级。

例：

- 雷纹淬膜丹：皮肤经验 +110，丹毒 +16，可能附加短期“皮膜灼痛”
- 玄铁锻骨丹：筋骨经验 +120，丹毒 +18，自然恢复短时下降
- 真火炼脏丹：脏腑经验 +130，丹毒 +22，失败时可能附加“脏腑暗伤”

#### 破限类

用于肉身阶位晋升，不是普通经验药。

例：

- 破限资材：按材料类型与最低品质判断，不绑定固定材料名
- 炼体丹药：按炼体丹的丹力归属与丹品判断，不绑定固定丹名

### 7.2 材料药性映射

炼丹的药性标签应从旧 `tempering_vitality` 等迁移为新五轨。

建议新药性：

```ts
body_skin
body_sinew_bone
body_organs
body_qi_blood
body_primordial_spirit
```

药性来源示例：

| 药性 | 典型材料 |
|---|---|
| 皮肤 | 雷击藤、风蚀鳞、寒铁砂、毒瘴花 |
| 筋骨 | 妖兽骨髓、玄铁精、地磁石、龙骨碎 |
| 脏腑 | 五行灵果、真火莲、寒肺草、土脾芝 |
| 气血 | 妖兽精血、血参、朱果、穴窍砂 |
| 元神 | 魂晶、清神芝、月华露、定魂木 |

### 7.3 炼丹意图解析

即兴炼丹中，玩家输入“淬体、炼体”时不应只随机落到一条轨道。应按关键词推断：

- “刀枪不入、皮膜、抗毒、药浴” -> 皮肤
- “骨、髓、力量、重压、玄铁” -> 筋骨
- “五脏、五气、真火、爆发、雷音” -> 脏腑
- “气血、精血、续航、寿元、法天象地” -> 气血
- “元神、神识、心魔、夺舍、渡劫” -> 元神

如果意图模糊，可以生成 `body_hybrid`，但数值效率低于专精丹。

## 8. 与现有架构的关系

### 8.1 当前架构可复用部分

以下模块可以复用：

- `condition` 持久状态：适合承载炼体轨道进度
- `ConditionOperation.advance_track`：可以扩展为支持新轨道
- `PillOperationExecutor`：可继续负责丹药推进与结算
- `AlchemyRecipeRules`：可继续从药性生成 `advance_track`
- battle-v5 modifier 系统：适合承载炼体派生加成
- display/combat adapter：适合从 cultivator + condition 派生最终面板

### 8.2 当前架构不支持或支持不足的地方

#### 不足 1：`ConditionTrackPath` 与旧五属性绑定

当前 `ConditionTrackPath` 是：

```ts
`tempering.${TemperingTrackKey}` | 'marrow_wash'
```

而 `TemperingTrackKey` 是：

```ts
'vitality' | 'spirit' | 'wisdom' | 'speed' | 'willpower'
```

这会把炼体继续锁死在五维属性语义上。

升级方案：

```ts
export type BodyCultivationTrackKey =
  | 'skin'
  | 'sinew_bone'
  | 'organs'
  | 'qi_blood'
  | 'primordial_spirit';

export type ConditionTrackPath =
  | `body.${BodyCultivationTrackKey}`
  | 'marrow_wash';
```

兼容期可以保留旧 `tempering.*` 读取，但新丹药、新 UI、新配置只写 `body.*`。

#### 不足 2：轨道奖励只能表达属性或灵根

当前 `TrackReward` 只有：

- attribute
- spiritual_root

无法表达“最大气血 +%”“控制抗性 +%”“战后恢复”。

升级方案：将轨道配置拆为“进度配置”和“效果配置”。

```ts
export interface BodyTrackConfig {
  key: BodyCultivationTrackPath;
  name: string;
  layerName: string;
  thresholdByLevel(level: number): number;
  effectsByLevel(level: number): BodyTrackEffect[];
  milestoneByLevel(level: number): BodyMilestone | null;
}
```

其中 `BodyTrackEffect` 不直接写入数据库，而是在 adapter 中转换为 battle-v5 modifier 或 condition hook。

#### 不足 3：battle-v5 缺少统一的炼体 adapter

当前 display adapter 从基础属性、功法、法宝构建 Unit。炼体如果继续散落在各处，会变成隐式逻辑。

升级方案：新增纯函数：

```ts
buildBodyCultivationModifiers(condition): AttributeModifierConfig[]
buildBodyCultivationRuntimeHooks(condition): BodyRuntimeHooks
getBodyCultivationSummary(condition): BodyCultivationSummary
```

`createDisplayUnitFromCultivator` 和 `createCombatUnitFromCultivator` 都通过同一套函数读取炼体 modifier，避免展示和战斗不一致。

#### 不足 4：状态 hook 目前主要围绕 condition status

皮肤抗侵蚀、筋骨伤势降级、气血战后恢复、元神抗心魔，都不是简单 battle modifier。它们需要接入不同生命周期：

- battle init
- damage taken
- battle settled
- natural recovery
- breakthrough failure
- dungeon event resolution

升级方案：增加炼体效果解析层，不把它伪装成 `ConditionStatusInstance`。

建议：

```ts
export interface BodyCultivationEffectHooks {
  onBattleInit?: ...
  onBattleSettle?: ...
  onNaturalRecovery?: ...
  onBreakthroughAttempt?: ...
  onDungeonEventResolve?: ...
}
```

首版可以只实现 display/combat modifier、自然恢复、突破失败三个接入点，后续再扩展事件。

#### 不足 5：前端只能显示 Lv 和进度

当前 UI 只展示 `Lv.x` 和 `progress / threshold`。这无法解释炼体为什么有用。

升级方案：新增 summary DTO 或共享派生函数，前端展示：

- 当前轨道等级
- 当前累计效果
- 下一级效果
- 5 级节点进度
- 肉身阶位
- 破限条件

## 9. 目标数据模型

### 9.1 condition 存储结构

建议目标结构：

```ts
export interface BodyCultivationState {
  version: 1;
  realm: BodyCultivationRealm;
  tracks: Record<BodyCultivationTrackKey, ConditionProgressTrack>;
  milestones: Partial<Record<BodyMilestoneKey, boolean>>;
  breakthrough?: {
    targetRealm: BodyCultivationRealm;
    progress: number;
    failedAttempts: number;
  };
}

export interface CultivatorCondition {
  tracks: {
    bodyCultivation: BodyCultivationState;
    marrowWash: ConditionProgressTrack;
  };
}
```

兼容期也可以先保持当前 `tracks.tempering` 字段，在 `normalizeCondition` 中派生出新结构。但目标上应避免长期双写。

### 9.2 新轨道 key

| 显示名 | 存储 key | track path |
|---|---|---|
| 皮肤 | `skin` | `body.skin` |
| 筋骨 | `sinew_bone` | `body.sinew_bone` |
| 脏腑 | `organs` | `body.organs` |
| 气血 | `qi_blood` | `body.qi_blood` |
| 元神 | `primordial_spirit` | `body.primordial_spirit` |

### 9.3 旧数据迁移

旧数据不建议丢弃。建议一次性迁移或 normalize 时懒迁移。

推荐映射：

| 旧轨道 | 新轨道 | 理由 |
|---|---|---|
| `tempering.vitality` | `body.qi_blood` | 旧体魄最接近生命和气血 |
| `tempering.spirit` | `body.organs` | 旧灵力最接近脏腑动力和神通承载 |
| `tempering.wisdom` | `body.primordial_spirit` | 悟性与元神识念最接近 |
| `tempering.speed` | `body.skin` | 身法与外护、反应、皮膜应激可近似承接 |
| `tempering.willpower` | `body.sinew_bone` | 意志承压与筋骨架构可近似承接 |

该映射不是世界观上的完美等价，但能最大限度保护玩家既有投入。迁移后 UI 应提示“旧炼体已重铸为新五层肉身根基”。

## 10. 运行时派生方案

### 10.1 纯函数入口

所有炼体效果必须由共享纯函数派生，禁止在 UI、服务端路由、战斗流程中各写一套。

建议新增：

```text
src/shared/lib/bodyCultivation/
  config.ts
  normalize.ts
  effects.ts
  summary.ts
  migration.ts
```

职责：

- `config.ts`：轨道、阈值、节点、阶位配置
- `normalize.ts`：默认值和边界
- `effects.ts`：把等级转成 modifier / hook
- `summary.ts`：给前端展示用
- `migration.ts`：旧 `tempering.*` 到新 `body.*`

### 10.2 battle-v5 接入

在 display/combat adapter 中挂载炼体 modifier。

示例：

```ts
const bodyModifiers = buildBodyCultivationAttributeModifiers(cultivator.condition);
for (const modifier of bodyModifiers) {
  unit.attributes.addModifier({
    ...modifier,
    source: {
      sourceType: 'bodyCultivation',
      carrierId: modifier.id,
    },
  });
}
```

首版建议只接 battle-v5 已有属性类型。若缺少某些属性，应先使用已有近似属性，不急于扩大战斗属性枚举。

推荐首版可落地属性：

- 最大气血
- 最大法力
- 防御 / 法防
- 暴击抗性
- 控制抗性
- 命中 / 闪避
- 治疗增幅

对于“战后恢复”“突破失败降级”等非战斗面板效果，用 hook 层实现。

### 10.3 condition 接入

`ConditionService` 需要读取炼体效果：

- 自然恢复时读取筋骨 / 气血恢复倍率
- 伤势写入时读取皮肤 / 筋骨降级概率
- 突破失败时读取筋骨 / 元神减免

不要把炼体效果转换为临时 `ConditionStatusInstance`。炼体是长期成长，condition status 是状态实例，两者语义不同。

### 10.4 炼丹接入

`AlchemyRecipeRules` 中旧 `tempering_*` 药性应迁移为新 `body_*` 药性。

`advance_track` 可以继续使用，但 track path 应为新 `body.*`。

示例：

```ts
{ type: 'advance_track', track: 'body.skin', value: 60 }
```

长期看，`advance_track` 应支持不同 track family：

```ts
type ConditionTrackPath =
  | BodyCultivationTrackPath
  | 'marrow_wash';
```

### 10.5 前端接入

前端不应自己计算复杂效果。应消费共享 summary：

```ts
getBodyCultivationSummary(condition)
```

展示结构：

- 肉身阶位：铁骨
- 五轨列表：皮肤 / 筋骨 / 脏腑 / 气血 / 元神
- 当前效果：例如“直接伤害减免 +6%，侵蚀抗性 +8%”
- 下一等级：例如“下一级：直接伤害减免 +0.6%”
- 下个节点：例如“Lv.15 解锁：外膜护体强化”
- 破限条件：例如“筋骨 Lv.8、皮肤 Lv.6、总等级 30”

## 11. 与突破系统的关系

炼体不应提高所有突破成功率，否则会与突破丹、感悟、修为进度抢定位。它应该影响“突破承压”和“失败后果”。

建议：

- 皮肤：降低雷劫 / 外邪侵蚀造成的伤势
- 筋骨：降低经脉、骨骼类失败惩罚
- 脏腑：降低燃烧气血或强行突破造成的内伤
- 气血：失败后保留更多 HP / 恢复更快
- 元神：降低心魔、夺舍、神魂动摇概率

大境界突破时，可以展示：

```text
肉身承压：
  筋骨：本次失败重伤概率 -12%
  元神：心魔滋生概率 -8%
  气血：失败后气血保留 +10%
```

这样炼体参与突破，但不直接变成另一个突破成功率堆叠器。

## 12. 与副本和连续作战的关系

炼体最适合在副本中体现价值。

建议副本事件接入：

- 皮肤：瘴气、毒雾、寒煞、魔气事件减免
- 筋骨：坠落、重压、近战硬抗事件减免
- 脏腑：火毒、五行冲突、爆发战事件减免
- 气血：连续战斗后恢复、低血探索风险降低
- 元神：幻境、魅惑、夺舍、心魔事件减免

副本 UI 可以在事件结算中提示：

```text
皮肤·铁膜生效：瘴气侵蚀降低，未生成“丹毒郁结”。
元神·身神相照生效：识破幻境，避免一次心神损耗。
```

这种提示比面板上的 `+1` 更能让玩家感知炼体价值。

### 13.1 角色面板

新增“肉身”区块，替换当前“洗髓与炼体”的简单列表。

展示：

- 总等级

## 14. 迁移与分期落地

### Phase 1：新五轨与展示落地

目标：不改 battle-v5 深层机制，先让数据和 UI 对齐。

任务：

- 新增 `BodyCultivationTrackKey`
- 新增新五轨配置和 summary
- `normalizeCondition` 支持旧轨到新轨迁移
- 前端“洗髓与炼体”改为“肉身炼体”
- 丹药显示新 track 名称

验证：

- 旧 condition 可正常 normalize
- 新五轨显示正确
- 旧炼体进度不会丢

### Phase 2：独立 modifier 接入

目标：解决“被突破属性覆盖”的根问题。

任务：

- 新增 `buildBodyCultivationAttributeModifiers`
- display adapter 接入炼体 modifier
- combat adapter 接入同一套 modifier
- 停止新炼体升级直接写 `Cultivator.attributes`
- 旧属性奖励不回滚，只是不再新增

验证：

- 突破前后炼体 bonus 仍存在
- 展示面板和战斗实际属性一致
- 炼体不污染 `Cultivator.attributes`

### Phase 3：节点和状态 hook

目标：让炼体进入玩法结算。

任务：

- 筋骨 / 气血接入自然恢复
- 筋骨 / 元神接入突破失败后果
- 皮肤 / 元神接入 dungeon 事件
- 实现 5 级节点

验证：

- 自然恢复测试
- 突破失败测试
- 副本事件测试

### Phase 4：肉身阶位和破限

目标：形成长期追求。

任务：

- 新增肉身阶位
- 新增破限条件
- 新增破限丹和材料需求
- 新增失败风险和保底

验证：

- 阶位晋升条件测试
- 破限失败不破坏状态一致性
- UI 正确展示缺失条件

### Phase 5：内容化和构筑化

目标：让体修玩法独立成型。

任务：

- 高阶材料按五轨投放
- 肉身神通需要脏腑 / 元神等级
- 副本炼体事件
- 体修排行榜或角色标签

## 15. 风险与控制

### 15.1 数值膨胀风险

风险：百分比收益如果无上限，会让炼体压过境界。

控制：

- 肉身阶位软上限
- 同类效果递减
- 关键质变设置冷却
- 不直接提高所有伤害和所有成功率

### 15.2 系统复杂度风险

风险：五轨、节点、阶位、丹药、hook 同时上线会过重。

控制：

- 分期上线
- Phase 1 和 Phase 2 只做数据、展示和 modifier
- hook 和破限后置

### 15.3 旧数据兼容风险

风险：旧 `tempering.*` 数据迁移错误导致玩家进度丢失。

控制：

- 先懒迁移，不立即删除旧字段
- migration 纯函数加测试
- UI 显示迁移后的新轨道
- DB 迁移前保留备份窗口

### 15.4 前后端展示不一致风险

风险：前端 summary 和战斗 adapter 分别计算，导致显示与实战不同。

控制：

- 所有 summary 和 modifier 来自 `src/shared/lib/bodyCultivation`
- 前端只调用共享函数
- 服务端战斗也调用同一函数

## 16. 推荐最终形态

最终炼体系统应形成以下体验：

玩家不再只看到“炼体·体魄 +1”。玩家会看到自己在修五层肉身：

- 皮肤让自己刀枪难伤、百毒不侵
- 筋骨让自己承重、恢复、濒死不倒
- 脏腑让自己爆发、承载肉身神通
- 气血让自己持久作战、越战越稳
- 元神让自己抵抗心魔、夺舍、渡劫排异

境界决定玩家站在哪个大层级。炼体决定玩家在这个层级里能不能硬抗、久战、爆发和承压。

这套设计既符合修仙小说的身体理论，也符合游戏系统架构：长期状态存在 `condition`，战斗属性由 battle-v5 adapter 派生，丹药通过 `advance_track` 推进，副本和突破通过 hook 读取炼体效果。它不是孤立玩法，而是连接炼丹、战斗、副本、突破和角色展示的第二成长骨架。
