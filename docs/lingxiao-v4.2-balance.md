# 凌霄剑宗 V4.2 平衡实测

## 测试口径

- 使用 battle-v5 实际投影、实际技能蓝耗、心法常驻属性与 AI 选招。
- 筑基初期、金丹圆满、化神圆满的心法等级分别为 25、60、100。
- 每个组合固定种子运行 100 局，并交替先手；无装备和灵根优势。
- 另以 1 级、180 级各 5 局执行边界冒烟。

## V4.2 机制基线

- 神通固定蓝耗，不再随境界变化。
- 神通伤害、护盾、治疗、状态强度和显式声明的持续时间由所属心法统一投影。
- 《一剑破妄》的驱散数量随《观微剑意》成长档由 1 个提高至 7 个，最终配置只保留普通整数。
- 剑痕与裂甲均为 3 层、3–6 回合状态；剑痕逐层增伤，裂甲逐层降低物防。
- 快剑每连续两次自身行动未造成直接伤害才衰减 1 点剑势；重剑以裂甲、护盾破裂和后发重击组织循环。
- 回燕代表构筑使用留痕，装配《剑荡山河》《藏锋听雷》《踏雪无痕》《剑破万法》；极势在裂甲不足 2 层时先补裂甲。
- 普通敌对神通的效果链统一要求命中；基础与重剑《藏锋听雷》改为自身蓄势，施展时登记必中、不可打断的后发《听雷》。
- 宗门节点卡片通过运行时 resolver 展示当前心法等级的实际数值，静态定义只保留机制摘要。

## 编译边界收尾

- 心法成长声明只存在于 sect authoring 阶段，最终投影会剥离全部宗门元数据，battle-v5 的 `BuffConfig` 只保留已结算数值。
- 编译顺序固定为“基础神通 → 流派变体 → 经脉节点 → 一次成长投影 → 展示分析”，不再用 `methodGrowthApplied` 保存编译阶段状态。
- 成长策略对 Effect 联合类型执行穷尽分类；新增 Effect 时必须显式决定其倍率、状态或固定数值语义。
- 分层属性修改器使用稳定标识，仅在层数变化时重挂载依赖层数的 modifier，避免随机标识改变固定种子结果。

## 门禁结论

成长、驱散数量、蓝耗、命中语义、状态覆盖、被动嵌套成长、动态节点文案、authoring 元数据剥离、36 节点运行时矩阵和 729 套组合编译均已进入自动化测试。严格 3×3 门禁也已改为默认执行。

当前固定种子平衡门禁仍未通过：只有筑基“追风 vs 后发”满足代表胜率与回合要求；其余代表对局及多数交叉对局越界，筑基和化神“回燕 vs 守山”仍为 0% 胜率。按本轮“保留九个流派基础神通倍率、只围绕指定身份节点以 5% 粒度校准”的边界，现有早期流派装配已经形成 0%–97% 的克制跨度，不能标记为平衡验收完成。继续收敛需要放开至少一项边界：代表装配/属性分配、流派基础神通数值，或接受更宽的交叉胜率区间。

严格门禁命令：

```bash
bunx vitest run src/shared/engine/sect/content/lingxiao/tests/balanceSimulation.test.ts
```

该命令当前会按上述实测差距失败，避免未达标数据被误标为平衡验收完成。
