# Creation-v2 数值平衡总方案与实施计划

## 1. 背景与目标

当前 creation-v2 架构已可运行，但数值层存在结构性失衡：
- 材料能量增长过快，导致高阶词缀解锁几乎常态化。
- 词缀池匹配过宽（tag OR），定向投入反馈不足。
- 词缀数量上限固定，预算溢出无法转化为稳定收益。
- 词缀强度缺少统一价值单位，不同效果类型难以等价比较。

本方案目标：
- 将造物体验从“可运行”升级为“可预测、可解释、可调参”。
- 在同境界基线下，将高品质造物对战节奏稳定在 5-8 回合。
- 建立可持续迭代的平衡框架（配置化+测试化+审计化）。

## 2. 设计原则

- 约束先行：输入边界必须明确，超限即失败。
- 预算闭环：总能量必须可追溯到每个来源和每次消耗。
- 分类配额：词缀分布由预算驱动，不依赖单一随机。
- 统一标尺：所有词缀效果映射到同一强度预算单位。
- 小步上线：每阶段可独立验收，失败可回滚。

## 3. 当前问题基线

### 3.1 材料投入
- 缺少强校验边界（1-6种材料、单材料数量1-3）作为统一入口约束。

### 3.2 能量预算
- 能量主要来源于线性公式，数量与品质收益过于陡峭。
- 阈值与预算量级不匹配，导致高阶解锁常驻。

### 3.3 词缀结构
- 词缀候选采用 OR 命中，池子过大。
- 当前核心策略是 core first + 固定最大数量，缺少预算驱动的槽位模型。

### 3.4 词缀数值
- 不同效果类型缺少统一价值换算，容易出现隐性超模或弱模。
- 部分能力标签与 battle-v5 伤害类型判定存在潜在错位风险。

## 4. 最终方案（目标态）

### 4.1 输入边界模型
- 强校验规则：
  - 材料种类数：1 <= kinds <= 6
  - 单材料数量：1 <= quantity <= 3
- 违反任一规则直接失败，不进入后续造物阶段。

### 4.2 能量预算模型 V2
- 单材料能量使用“品质权重 * 数量递减函数 + 类型奖励”。
- 公式示意：
  - E_i = W_q * sqrt(quantity_i) + B_type
- 总能量引入结构奖励：
  - E_total = sum(E_i) + B_diversity + B_coherence
- 预算审计要求：
  - total = reserved + spent + remaining 始终成立。

### 4.3 词缀分配模型
- 保留 core first。
- 将后续词缀改为“预算分层槽位 + 分类配额 + 加权采样”。
- 高阶词缀（signature/synergy/mythic）不再仅靠极低权重抽中。

### 4.4 强度预算单位（PBU）
- 引入 PBU（Power Budget Unit）用于统一效果强度。
- 各词缀参数由 PBU 反推，避免跨类型不可比。
- 伤害/治疗/护盾/控制等统一纳入同一预算框架。

## 5. 分阶段实施计划

### Phase 1（立即实施）
- 输入强校验落地（1-6种材料，单材料1-3）。
- 能量公式V2落地（递减数量收益 + 结构奖励）。
- 解锁阈值与保留能量重标定。
- 更新单元测试与集成测试基线。

### Phase 2
- 候选池打分化（替代纯 OR 扩池）。
- 配额化抽取（分类保底与上限）。
- 高阶词缀命中率目标化配置。

### Phase 3
- PBU系统落地。
- 战斗结果回归校准（TTK、EHP、资源压制曲线）。
- 开放调参面板与自动化压测报告。

## 6. 验收标准

- 输入边界：所有越界输入都能在流程入口失败并给出明确原因。
- 预算闭环：所有造物会话满足能量守恒方程。
- 分布目标：词缀数量与类别分布符合预算层级设计。
- 战斗目标：同境界高品质造物的中位TTK位于3-5回合。
- 可观测性：每次造物都可输出决策轨迹与预算审计。

## 7. 风险与回滚

- 风险1：旧测试大量依赖固定能量值。
  - 方案：同步更新测试基线并增加性质测试（守恒与单调性）。
- 风险2：阈值重标后低投入体验变差。
  - 方案：保底核心词缀与低投入保护阈值。
- 风险3：改动链路较长。
  - 方案：按阶段开关与灰度验证，必要时可退回旧预算公式。
