# 造物引擎 v2 词条三角重构设计（梦幻西游风格）

## 1. 目标

基于《梦幻西游》的设计理念，重构 Creation-V2 词条库，使 **Skill / Gongfa / Artifact** 三个维度拥有清晰边界、不同的稀有感来源，以及稳定的三角联动关系。

本次设计不考虑历史兼容与历史包袱，按未上线系统进行彻底重构。
**注意：重构的是 Creation-v2 路径在 `engine/creation-v2`，请勿读取 Creation-v1的代码!!!**

## 2. 设计原则

### 2.1 平衡三角

- **Skill**：负责“出手惊艳感”，词条价值集中在本次施法。
- **Gongfa**：负责“流派成立感”，词条价值集中在全局规则与倍率。
- **Artifact**：负责“装备出货感”，词条价值集中在面板底力、受击反馈、保命与对策。

三者都可以出极品，但极品的理由必须不同：

- **Skill 的极品**：这一招是否打出神来之笔。
- **Gongfa 的极品**：这一套 build 是否真正成立。
- **Artifact 的极品**：这一件货是否值得追求与交易。

### 2.2 梦幻式稀有感

词条体系不追求“所有东西都靠极品”，而追求：

1. **基础词条**保证可用；
2. **方向词条**保证成型；
3. **稀有词条**提供明显战术身份；
4. **传说词条**改写战斗决策，而不是只给更大数值。

### 2.3 代码层硬边界

- **Skill** 不允许常驻属性，不承担长期规则层。
- **Gongfa** 不提供固定面板，不承担受击型防御特效。
- **Artifact** 不定义主流派规则，不承担主动施法型爆发逻辑。

## 3. 词条池结构

### 3.1 Skill

#### A. 核心池（约 50%）

职责：保证技能不废，专注本次施法直接成立。

允许内容：

- 伤害提高
- 治疗提高
- 命中提高
- 消耗降低
- 小概率附加灼烧 / 冰缓 / 中毒 / 破甲
- 本次技能附带少量回血 / 回蓝 / 护盾

建议 GE 类型：

- `damage`
- `heal`
- `mana_burn`
- `dispel`
- `apply_buff`
- `attribute_stat_buff`（仅短效）

约束：

- 只能影响本次施法或短持续结果；
- 不通过长持续 buff 模拟长期被动系统。

#### B. 变招池（约 35%）

职责：让同一技能长出不同战术身份。

允许内容：

- 单体变多段
- 群攻强化主目标
- 控制技能提高命中
- 治疗技能附带净化
- 输出技能附带破盾 / 驱散
- 本次技能暴击 / 爆伤 / 斩杀条件强化

建议 GE 类型：

- `percent_damage_modifier`
- `cooldown_modify`
- `apply_buff`
- `tag_trigger`
- `dispel`

#### C. 稀有技池（约 15%）

职责：制造“神技感”。

典型模板：

- **引燃**：命中灼烧目标时立即引爆一次灼烧
- **封喉**：目标受控时触发终结增伤
- **逆脉**：命中目标有概率延长对方技能冷却
- **魂伤**：真实伤害，无视防御

规则：

- 每个 Skill 最多出现 1 条稀有/传说词条；
- 稀有词条必须围绕“本次施法”做文章；
- 不允许依赖多回合持续规则层。

### 3.2 Gongfa

#### A. 根基池（约 45%）

职责：像修炼 / 经脉一样给整套 build 定底子。

允许内容：

- 攻 / 防 / 血 / 速百分比
- 暴击加成
- 治疗加成
- 控制命中
- 控制抗性
- 通用增伤 / 减伤

建议 GE 类型：

- `attribute_modifier`（百分比 / ADD）
- `percent_damage_modifier`

#### B. 流派池（约 40%）

职责：定义“这套到底怎么玩”。

允许内容：

- 火 / 冰 / 雷 / 毒等元素专精
- 对灼烧 / 冰缓 / 中毒 / 受控目标增伤
- 暴击回蓝、低蓝增伤
- 首回合强化、低血强化、护盾存在时强化
- 状态延长、DOT 放大、控制链强化

建议 GE 类型：

- `percent_damage_modifier`
- `resource_drain`
- `cooldown_modify`
- `apply_buff`（仅作为规则容器，避免滥用）

约束：

- 规则应优先直接挂在 Listener 上；
- 仅在 battle-v5 必须通过 buff 承载时才使用长期 buff 包装；
- 不允许出现受击型防守定位。

#### C. 稀有诀池（约 15%）

职责：制造“门派真传感”。

典型模板：

- **焚天诀**：火系技能命中灼烧目标时再提高最终伤害
- **寒魄诀**：攻击冰缓目标附带最大生命比例伤害
- **轮回诀**：技能命中后有几率减少随机技能 CD
- **无相诀**：根据当前最高副属性切换强化方向

规则：

- 稀有 Gongfa 数量最少；
- 单条词条的 build 影响最大；
- 稀有 Gongfa 决定“流派身份”，不直接替代 Skill 出手逻辑。

### 3.3 Artifact

#### A. 面板池（约 55%）

职责：决定这件装备是不是好货。

允许内容：

- 武器：物攻 / 法攻 / 穿透
- 护甲：物防 / 法防 / 气血
- 饰品：速度 / 暴击 / 命中 / 抗性 / 控制属性

建议 GE 类型：

- `attribute_modifier`（固定值 / FIXED）
- `random_attribute_modifier`（固定值池）

#### B. 防守池（约 30%）

职责：提供容错、反制、保命与对策。

允许内容：

- 受击减伤
- 低血护盾
- 濒死保命
- 被控时解控 / 免疫
- 被暴击后回盾 / 反伤 / 反制
- 特定元素 / 通道减伤

建议 GE 类型：

- `shield`
- `magic_shield`
- `reflect`
- `damage_immunity`
- `death_prevent`
- `percent_damage_modifier`（仅 reduce）

约束：

- Listener 以 `OWNER_AS_TARGET` 为主，必要时允许 `GLOBAL`；
- 不允许主动出手型攻击爆发逻辑；
- 不允许承担主流派定义。

#### C. 稀有宝池（约 15%）

职责：制造“极品法宝感”。

典型模板：

- **金甲**：受击概率大幅度减伤
- **护命**：首次濒死保留 1 血并获得护盾
- **太虚镜**：特定元素伤害概率完全免疫

规则：

- Artifact 的传说词条最值钱；
- 稀有价值来自“底线能力 + 对策能力”，而不是 build 主轴。

## 4. 稀有度与生成规则

### 4.1 四层稀有度

每个维度都采用相同层级，但承担的价值不同：

1. **基础词条**：保证能用
2. **优秀词条**：开始成型
3. **稀有词条**：产生战术身份
4. **传说词条**：改写决策

### 4.2 出货结构

建议每件产物：

- 固定 1 条核心词条
- 其余词条根据能量预算进行解锁
- 然后根据match匹配到可选词条
- 然后进行抽取，消耗能量预算点数
- 通过点数消耗来控制词条的最终的出货数量

### 4.3 配比倾向

- **Skill**：稀有率最高，承担爽感来源
- **Gongfa**：稀有率最低，但单条 build 价值最高
- **Artifact**：传说率最低，但单件价值最高

## 6. battle-v5 映射约束

### 6.1 Skill

- 顶层效果应以即时 GE 为主；
- 移除 `attribute_stat_buff` 如果需要增加属性的buff，直接定义 `apply_buff` ；
- 禁止通过长时 `apply_buff + nested listener` 模拟长期被动系统。

### 6.2 Gongfa

- 以 `OWNER_AS_CASTER` / `GLOBAL` 为主；
- 可定义规则、倍率、资源节奏；
- 不允许顶层 `OWNER_AS_TARGET` 防守域。

### 6.3 Artifact

- 以 `OWNER_AS_TARGET` / `GLOBAL` 为主；
- 固定值面板必须保留在 Artifact 域；
- 防御、反伤、保命、减伤均归 Artifact 域。

## 7. 重构判定标准

重构完成后，应满足以下标准：

1. 玩家能直觉区分：
   - Skill 是“招”
   - Gongfa 是“路”
   - Artifact 是“货”
2. 任意极品词条出现时，玩家都能明确知道它为什么值钱。
3. 三个维度缺一不可，但不互相抢职责。
4. build 的强度来自三维联动，而不是单维度堆满万能神词条。
