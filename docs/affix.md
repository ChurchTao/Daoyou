在 RPG 游戏设计中，明确主动技能 (Skill)、被动功法 (Gongfa) 和 装备法宝 (Artifact) 的边界是构建深度战斗系统的核心。目前的词缀定义确实存在逻辑重叠

  基于修仙世界观和战斗引擎 v5 的底层逻辑（GE/Listener 机制），参考《梦幻西游》和《Dota 2》，我为你提供以下设计边界建议：

  1. 核心定位与设计边界

  ┌──────────┬─────────────────────────────────────┬─────────────────────────────────────┬─────────────────────────────────────────┐
  │ 维度     │ 主动技能 (Skill/神通)               │ 被动功法 (Gongfa/心法)              │ 装备法宝 (Artifact/器物)                │
  ├──────────┼─────────────────────────────────────┼─────────────────────────────────────┼─────────────────────────────────────────┤
  │ 功能定位 │ 变数与决策 (The Move)               │ 根基与流派 (The Engine)             │ 数值与补丁 (The Tool)                   │
  │ 触发时机 │ 玩家/AI 选定释放 (Active)           │ 持续存在或特定条件触发 (Passive)    │ 常驻属性或受击/外部反馈 (Reactive)      │
  │ 资源消耗 │ 消耗灵力 (MP)、回合次数、CD         │ 无消耗，或作为某种机制的“调节器”    │ 装备位限制、祭炼值 (Energy)             │
  │ 设计边界 │ 产生即时影响 (伤害、控制、瞬间爆发) │ 改变规则与倍率 (属性转化、机制联动) │ 提供基础属性与防御余地 (面板、受击反馈) │
  └──────────┴─────────────────────────────────────┴─────────────────────────────────────┴─────────────────────────────────────────┘

  2. 深入参考：成熟游戏逻辑

  《梦幻西游》的启示：
   * 技能：龙卷雨击（纯输出）、推气过宫（纯治疗）。核心是消耗 MP 换取即时效果。
   * 功法 (修炼/经脉)：提升的是“物伤结果+10%”、“封印命中+5%”。它不直接造成伤害，而是放大技能的效果。
   * 法宝：
       * 被动法宝（金甲仙衣）：受击时 20% 概率减伤。这是防御性的、随机的补偿机制。
       * 主动法宝（无魂傀儡）：特定的战术干扰。

  《Dota 2》的启示：
   * 技能：英雄的独特性来源（黑洞、变羊）。
   * 天赋/被动：定义英雄的成长方向（攻击距离、减 CD）。
   * 道具 (Artifact)：弥补短板。法师身脆买“绿杖/跳刀”；物理流买“大炮/金箍棒”。法宝应该用来补足修仙者在特定对局中的短板。

  3. 基于 v5 引擎的具体设计方案

  为了区分这三者，建议在 AffixDefinition 的设计上采用不同的 GE 类型 和 Listener 作用域：

  A. 主动技能 (Skill)：侧重 Direct Effects
   * 设计规范：严禁提供常驻属性。
   * 推荐 GE 类型：damage, heal, dispel, apply_buff (短期强力状态)。
   * 建议：
       * 词缀应集中在改变目标数量、附加异常状态、爆发倍率。
       * 例如：skill-core-damage（基础伤害）、skill-suffix-burn-dot（施法后挂火）。

  B. 被动功法 (Gongfa)：侧重 Systemic Mechanics & Multipliers
   * 设计规范：定义修仙者的“流派”。
   * 推荐 GE 类型：attribute_modifier (百分比加成), percent_damage_modifier, cooldown_modify。
   * Listener 逻辑：应使用 OWNER_AS_CASTER（当我施法时...）或全局机制。
   * 建议：
       * 属性转化：如“将 10% 的神识转化为法攻”。
       * 机制增强：如“所有火系技能有 50% 概率多叠一层灼烧”。
       * 资源回补：如“每次造成暴击时回复 2% 灵力”。

  C. 装备法宝 (Artifact)：侧重 Fixed Stats & Reactive Triggers
   * 设计规范：提供面板底力，并在“受挫”时提供反馈。
   * 推荐 GE 类型：attribute_modifier (固定值加成), shield, reflect, damage_immunity。
   * Listener 逻辑：应侧重 OWNER_AS_TARGET（当我受击时...）或回合开始/结束。
   * 建议：
       * 基础面板：武器加攻，防具加防。
       * 生存补偿：受击护盾、濒死保命、反伤震慑。
       * 战术补丁：增加固定数值的法穿、忽视防御、闪避。
