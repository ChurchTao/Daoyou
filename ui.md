已实现核心系统（角色创建、战斗、演化），现在需要一个**以文字为核心、水墨风格、移动端优先、沉浸感强**的 UI 架构。下面我为你设计一套**纯文字 + 极简排版 + 氛围营造**的界面方案，完全适配手机屏幕，同时在 PC 上优雅降级。

---

## 🎨 整体设计原则

| 原则           | 说明                                                 |
| -------------- | ---------------------------------------------------- |
| **文字即界面** | 所有信息通过排版、符号、留白呈现，不用图标或图片     |
| **水墨意境**   | 使用中文标点、古风分隔符、留白呼吸感                 |
| **移动端优先** | 单列布局，操作区域在屏幕下半部（拇指热区）           |
| **状态清晰**   | 当前角色/背包/战斗等模块一目了然                     |
| **沉浸叙事**   | 界面语言采用修仙语境（如“道友请留步”而非“欢迎回来”） |

## 📐 响应式适配（PC 兼容）

- **移动端**：单列，按钮大，输入框高
- **PC 端**：
  - 内容居中，最大宽度 600px
  - 字体稍大（16px+）
  - 按钮横向排列（如 [挑战] [分享] 并排）
  - 保留所有文字氛围元素

## 🛠️ 技术实现建议

- **字体**：标题、UI中使用 Ma Shan Zheng, 内容区域使用 ZCOOL XiaoWei, 字体都已经引入
- **排版**：
  - 行高 1.6
  - 段间距用 `<br>` 或 `margin`
  - 重要信息加粗（如角色名）
- **交互**：所有按钮用 `<button>`，确保无障碍访问

## 🧩 Ink 组件体系

| 组件 | 用途说明 | 备注 |
| --- | --- | --- |
| `InkButton` | 方括号文字按钮，提供 `default/primary/secondary` 三种语气 | 用 `[文案]` 强调文字交互 |
| `InkBadge` | 徽记，用于境界/品阶，支持 tone 与 `tier`（凡/灵/玄/真/地/天/仙/神） | 品阶统一展示为 `「品阶」内容` |
| `InkTag` | 行内标签，标注元素/状态（如 `木·控制`） | `tone=good/bad/info` 控制色彩 |
| `InkStatRow` | 属性行组件，展示基础值 → 最终值及分解 | 适合体魄/灵力等数值 |
| `InkList/Item` | 纯文字列表容器，内建主标题/Meta/描述/操作区 | 所有清单（装备/命格/榜单）统一使用 |
| `InkInput` | 单行或多行文字输入，保留 Cmd/Ctrl+Enter 快捷提交 | 默认透明背景，强调文字为主体 |
| `InkNotice` | 空状态/提示文案组件，tone 控制语气 | 代替 `empty-state` 等临时类 |
| `InkStatusBar` | ❤️/⚡️/寿元等状态条，支持横排或堆叠 | 首页与道身页共用 |
| `InkToastHost` | 浮层提示，“法宝灵性已调顺”等反馈统一从此处弹出 | 文案遵循修仙语境 |
| `InkDialog` | 遮罩对话框，承担“以此真身入世？”等确认操作 | 默认按钮 `[罢][允]` 可自定义文案 |
| `InkPageShell` | 页面框架，负责标题/返回/底部导航与公用留白 | `statusBar`、`toolbar`、`footer` 插槽 |

- 真：`var(--tier-zhen)`（0xF97316）
- 地：`var(--tier-di)`（0xEA580C）
- 天：`var(--tier-tian)`（0xCA8A04）

```

```

- 警告：`请选择 3 个先天气运`
- Dialog 示例：
  - 标题：`以此真身入世？`
  - 内容：列出 `姓名` / `境界` / `灵根`
  - 按钮：`[再想想] [入世]`

## ✅ 总结：这套 UI 的优势

1. **零美术资源**：全靠文字排版与符号
2. **强沉浸感**：语言、符号、留白均服务修仙世界观
3. **移动端友好**：操作区域集中，文字大小适中
4. **扩展性强**：新增系统（如宗门、丹药）可沿用相同模式
5. **开发成本低**：几十行 CSS + 语义化 HTML 即可实现
